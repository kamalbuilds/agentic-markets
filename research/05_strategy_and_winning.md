## WHAT MAKES JUDGES PICK WINNERS

1. **Working demo beats everything** - Not slides. Real transactions.
2. **Fun beats complex** - Make judges smile
3. **"Why blockchain?"** - Must have obvious answer
4. **Consumer-friendly UX** - Blockchain should be invisible
5. **Cross-domain innovation** - Combine unexpected things
6. **Narrative fit** - 2026 = autonomous agent commerce

### Judging Dimensions
- Innovation, Feasibility, Execution, Integration, Validation, Pitch
- Sponsor judges are employees who want THEIR tech used meaningfully

## RECOMMENDED TECH STACK

```
Frontend:     Next.js 14+ (App Router) + TypeScript
Styling:      Tailwind CSS + shadcn/ui
Web3:         viem + wagmi v2 + RainbowKit
Contracts:    Solidity + Foundry
Boilerplate:  Scaffold-ETH 2
AI Agents:    ElizaOS (agent logic) + Vercel AI SDK (frontend)
Deployment:   Vercel (frontend) + testnets
```

**Why viem > ethers.js**: Type-safe, tree-shakeable, faster, modern standard
**Why shadcn/ui**: Source code ownership, beautiful, accessible, zero design skill needed
**Why ElizaOS**: Purpose-built for autonomous blockchain agents, TypeScript, $20B+ ecosystem
